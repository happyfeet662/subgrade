書き出し

書き出し設定の正解
YouTube最適化・H.264/H.265・ビットレート

せっかく綺麗に編集しても、書き出しで台無しに——「アップしたら眠くなった」「ブロックノイズが出た」の多くは設定のせいです。 Premiere の書き出しを“もう迷わない早見表”にまとめました。コピペ感覚で使えます。

書き出し・エンコード 読了 約12分 2026.07.12
まず結論（YouTube向け） 形式 H.264 / MP4、ビットレートは VBR・2パス、 1080pならターゲット8・最大12 Mbps、4Kならターゲット40・最大45 Mbps、 音声はAAC・320kbps。まずはこれで間違いありません。以下で理由と応用を解説します。

H.264 と H.265、どっちを選ぶ？

どちらもYouTubeは対応しています。違いは「速度・互換性・ファイルサイズ」のトレードオフです。

H.264H.265（HEVC）
エンコード速度速い遅い（負荷大）
互換性ほぼ全環境で再生可古い環境で不可の場合あり
同画質のファイルサイズ大きめ小さい（約半分）
向いている用途YouTube・納品・汎用（基本これ）4Kの保存・回線が細い環境・容量節約

迷ったら H.264。「4Kで容量を抑えたい」「配布先が最新環境と分かっている」ときだけ H.265 を検討します。

ビットレート早見表（YouTube目安）

ビットレートは「1秒あたりのデータ量」。高いほど高画質・大容量です。YouTube推奨をベースにした実用値がこちら。

解像度 / fpsターゲット最大
1080p / 24〜308 Mbps12 Mbps
1080p / 6012 Mbps18 Mbps
4K / 24〜3040 Mbps45 Mbps
4K / 6055 Mbps66 Mbps

YouTubeはアップロード後に再エンコードします。そのため「少し高め」に出しておくと、再エンコードの劣化を見越して結果がきれいになります。とはいえ上げすぎはアップロード時間の無駄。上の“最大”を上限の目安にしましょう。

VBR 1パス vs 2パス —— 2パス一択の理由

ビットレートの割り当て方には CBR（固定）と VBR（可変）があり、動画では VBR が基本。 さらに VBR には1パスと2パスがあります。

  • VBR 1パス：一度で書き出し。速いが、動きの激しい場面にビットレートを賢く配れない。
  • VBR 2パス：1回目で映像を解析し、2回目で動きの多い所へ多く、静かな所へ少なく配分。同じ容量でも画質が安定。

書き出し時間は約2倍になりますが、複雑なモーションでの差は歴然。納品・作品は迷わず2パスにしましょう。

ハードウェア vs ソフトウェアエンコード

書き出し画面のエンコード設定で選べます。

  • ハードウェアエンコード（GPU）：対応GPUがあれば大幅に高速。日常のYouTube書き出しはこれで十分。
  • ソフトウェアエンコード（CPU）：遅いが、細部の画質はわずかに有利とされる場面も。GPU非対応の古いマシンや、最高画質を追う納品で。

古いシステムや一部設定ではハードウェアが使えず、自動的にソフトウェアに切り替わることがあります。基本はハードウェアを選び、速く回すのが実用的です。

音声設定を忘れない

映像に気を取られて音声を放置しがち。AAC / 320 kbps / 48kHz / ステレオを選べば、YouTubeで音がこもりません。ナレーションや音楽の印象が変わるので侮れません。

プリセットを保存して毎回使い回す

上の設定を一度作ったら、書き出し画面でプリセットとして保存。次回からはワンクリックで同じ設定を呼び出せます。 「YouTube-1080p」「YouTube-4K」などと名前を付けておくと、書き出しの度に悩む時間がゼロになります。効率化の観点はショートカット＆自動化の記事もどうぞ。

つまずきポイント ・アップ後に眠い/ブロックノイズ → ビットレートを上げ、VBR2パスに。
・音が小さい/こもる → 音声がAAC 320kbpsか確認。ラウドネスも要チェック。
・書き出しが極端に遅い → ハードウェアエンコードになっているか、2パスで倍時間かかっていないか。
・色が変わる → 書き出しの色空間・タグ。姉妹記事のカラーマネジメント解説を参照。

よくある質問

結局いちばん無難な設定は？

H.264 / MP4 / VBR 2パス、1080pならターゲット8・最大12Mbps、音声AAC 320kbps。これが「迷ったらこれ」の鉄板です。プリセット化して使い回しましょう。

ビットレートを上げれば上げるほど綺麗になる？

ある点を超えると見た目はほぼ変わらず、ファイルとアップロード時間だけが増えます。YouTubeは再エンコードするので、早見表の“最大”あたりが実用的な上限です。

H.265で出したらYouTubeで再生できないと言われた。

視聴者の環境（古い端末・ブラウザ）でHEVC非対応の場合があります。配布先が不特定多数なら H.264 が安全です。H.265は保存用・限定配布向けと考えましょう。

Instagram や TikTok も同じ設定でいい？

コーデック（H.264/MP4）は共通でOKですが、解像度・アスペクト比が縦（9:16）になります。ビットレートは1080p基準で問題ありません。縦動画のテンプレ化は.mogrtの記事が役立ちます。

まとめ

書き出しは「H.264/MP4 → VBR2パス → 解像度に合わせたビットレート → 音声AAC320k → プリセット保存」で完成です。 一度テンプレを作れば、次からは考えずにワンクリック。作品のクオリティを最後の1工程で落とさないよう、ここだけは押さえておきましょう。
